How to export from Google Search Console

  1. Go to search.google.com/search-console
  2. Select your property (kloudesim.com)
  3. Click Performance in the left sidebar
  4. Set date range (last 28 days recommended)
  5. Make sure Queries and Pages are toggled on
  6. Click the Export ↓ button → Download CSV
  7. Import that file here — previous data is saved automatically for trend tracking

Each import saves the current data as "previous" so the next import shows ▲/▼ trends. Data is stored in your browser (localStorage) — nothing is sent anywhere.

How To Use the SEO Agent

What this agent does

The KloudeSIM SEO Agent generates programmatic, SEO and GEO-optimised landing pages for every destination you sell eSIMs to. Each page is built from your real pricing data — never invented — and includes an honest Holafly/Airalo comparison that makes it more likely to be cited by AI assistants (Google AI Overviews, Perplexity, ChatGPT).

One agent run produces a complete page: title, meta description, pricing table, competitor comparison, coverage section, install guide, FAQ, and JSON-LD schema. The humanising pass removes AI tells so the copy reads like it was written by a person.

Setup (one-time)

  1. Install Claude Code: npm install -g @anthropic-ai/claude-code
  2. Authenticate: claude and follow the login prompt
  3. Clone the repo and cd into it
  4. Replace placeholder prices in data/kloudesim-pricing.csv with your real plans
  5. Update source URLs in data/competitor-pricing.csv, then run /competitor-diff to pull live prices

Commands

/generate-destination <country>
Generate a single destination page interactively. Claude reads your pricing data, writes the full page, runs a humanising pass, self-checks it, and saves to output/esim-{slug}.md.
Example: /generate-destination Japan
/batch-generate [region]
Generates all missing destination pages in one run. Shows you the to-do list before starting. Optionally filter by region (Asia, Middle East, Europe). Each page gets its own full workflow — no copy-paste templating.
Example: /batch-generate Middle East
/competitor-diff
Fetches current prices from Airalo and Holafly for all tracked destinations, diffs against your CSV, updates it, and reports which pages have stale competitor data that needs regenerating.
Run weekly to keep comparison tables current

Headless batch (cron / GitHub Actions)

Run python scripts/generate_pages.py without opening Claude Code. Each destination gets a fresh Claude session so the humanising pass is never shortcut.

# All missing pages python scripts/generate_pages.py # Filter by region python scripts/generate_pages.py --region "Middle East" # Specific destinations only python scripts/generate_pages.py --only japan turkey # Force regenerate even if file exists python scripts/generate_pages.py --force

Rankings — Trend Arrows

Every time you import a GSC CSV, the previous import is automatically saved in your browser. The next import shows ▲ / ▼ deltas next to each position so you can see week-on-week movement at a glance. Nothing is sent to any server — data lives in localStorage.

To track trends: export and import your GSC CSV once a week. The dashboard does the rest.

Rankings — Quick Wins ⚡

Keywords ranked position 4–15 with 500+ impressions are flagged as Quick Wins. These pages are already visible to searchers but not yet winning the click. A targeted content update — stronger title, more specific FAQ, fresher competitor comparison — is usually enough to push them onto page 1.

Use the ⚡ Quick Wins only filter in the Rankings view to isolate these keywords.

Rankings — CTR vs Expected

Each keyword shows your actual CTR compared to the industry average for that position. If a keyword shows ▼ below expected, your title or meta description is underperforming — searchers are seeing your page but not clicking it.

11–20~1.5%Focus on moving to page 1 first, then fix CTR

Page generation rules (hard limits)

  • Never fabricates data. If a price, network name, or coverage fact isn't in your CSV, the agent flags it as [NEEDS DATA: x] rather than inventing it.
  • UK English throughout. Customise, organise, travelling.
  • Competitor prices carry a date. Data older than 30 days triggers a stale warning and blocks publishing.
  • One page per destination per run. Batches call the skill once per country — no templating across destinations.
  • Self-check before saving. Title ≤ 60 chars, meta 140–155 chars, one H1, comparison table present, FAQPage + Product JSON-LD valid, 3–5 internal links.

Publishing to your CMS

Generated pages land in output/esim-{slug}.md with full YAML frontmatter. The frontmatter includes title, meta_description, slug, country, region, last_updated, and lowest_price_gbp — everything a headless CMS (Contentful, Sanity, WordPress REST API) needs to ingest automatically.

Add a /publish command or CI step that reads output/*.md and pushes via your CMS API to complete the pipeline.
